1.什么是类?

  • 类就是用于描述对象的共性特征，主要用于描述对象的属性和行为。

2. 类与对象的关系

  • OC中的类好比生活当中的图纸，用来描述一些事物。也就是说，要想创建对象，必须先有类。

  • OC中的对象好比根据图纸创建出来的东西，是类的具体存在。因此，面向对象解决问题应该是先考虑需要设计哪些类，再利用类创建多少个对象。

3. 如何设计一个类

  • 生活中描述事物无非就是名称、属性、行为。

    • 如：人有名字，有身高、体重等属性，有说话、打架等行为。
      事物名称（类名）：人（Person）
      属性 ：身高（height）、年龄（age）
      行为（功能）：跑（run）

  • OC 中用类来描述事物也是如此

    • 事物名称：类名。
    • 属性：对应类中的成员变量。
    • 行为：对应类中的成员方法。

  • 定义类其实在定义类中的成员（成员变量和成员方法）。

4. 定义类

  • C 语言中函数分为声明和实现，OC 中定义一个类也分为声明和实现，也就是说，以后我们在 OC 中定义类，就是在写类的声明和实现。

  • 编写类的声明：

    • : NSObject 为了让我们的类具备创建对象的能力, 也就是可以使用new方法。

    • 以@interface 开头，以@end 结尾，然后 class name 对应的地方写上事物名称，也就是类名即可。

  • 类名的首字母必须大写。
  • 属性只能写在@interface 和@end 之间的大括号中。
  • 声明一个类的目的就是为了告诉系统，我们这个类有哪些属性跟行为。
  • 注意：编写 OC 类属性的时候，建议将所有属性的名称前面都加上_

  • 编写类的实现

    • 以@implementation 开头，以@end 结尾，然后在 class 对应的地方写上声明时声明的类名，必须和声明的类名一模一样。

5. 通过类来创建对象

  • 在 OC 中想要通过一个类来创建一个对象，必须给类发送一个消息（好比 C 语言中调用方法一样）。
  • 在 OC 中调用方法称为发送消息。
  • 发送消息：
    • 在 OC 中只要想要发送消息就先写上 [类名称/对象名称 方法名称];

  • 发送什么消息（调用什么方法）可以创建一个对象呢？

    • new方法
    • 类名 *p = [类名 new];

  • 只要通过一个类调用类的 new 方法，也就是给类发送一个叫做 new 的消息之后，系统内部就会做三件事：

    1. 为类创建创建出来的对象分配存储空间。
    2. 初始化类创建出来的的对象中的属性。
    3. 返回类创建出的对象对应的地址。
  • 如果给一个指针保存某个对象的的地址，那么我们就称这个指针为某个类型的对象。
    • eg. 利用 Iphone 类型的指针保存了 Iphone对象的地址，那么我们就称 Iphone 类型的指针 p 为 Iphone 对象。
  • OC 中的类其实本质就是一个结构体，所以 p 这个指针其实就是指向了一个结构体。
