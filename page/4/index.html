基本概念

1.

  • int main(a,b){}
    a : 启动时，外部给程序传递的参数个数。
    b : 分别保存各个参数。

2.

  • OC 是兼容 C 的。所以可以在 OC 程序中编写 C 语言代码，并且可以将 C 语言的源文件和 OC 的源文件组合在一起生成可执行文件。
  • OC 中跟C 中 printf函数功能相同的方法 NSLog();。
  • NSLog 是 OC 向控制台输出信息的方法。
  • 格式：NSLog(@”字符串”);。
  • NSLog()需要import<Foundation/Foundation.h>。

3.

  • Foundation.h 被称为主头文件，头文件中又拷贝了该工具箱中所有工具的头文件，我们只需要导入头文件就可以使用该工具箱中的所有的工具，避免了每次使用都要导入一个相应的头文件。
  • import 的作用跟 include 一样，是将右边的文件拷贝到当前 import 的位置。
  • import 的优点：自动防止重复拷贝。
  • import<>与 import””区别：
    • import<>:先编译器环境找，最后在系统环境找。
    • import””:先在当前文件找，然后在编译器环境找，最后在系统环境找。

#4.

  • printf 跟 NSLog 的区别：
    • NSLog 会自动换行。
    • NSLog 在输出内容时会附加一些系统信息。
    • NSLog 跟 printf 接收的参数不一样。

Homebrew/Homebrew Cask 常用命令汇总

hexo

npm install hexo -g #安装
npm update hexo -g #升级
hexo init #初始化
简写

hexo n “我的博客” == hexo new “我的博客” #新建文章
hexo p == hexo publish
hexo g == hexo generate#生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy#部署

服务器

hexo server #Hexo 会监视文件变动并自动更新，您无须重启服务器。
hexo server -s #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP

hexo clean #清除缓存 网页正常情况下可以忽略此条命令
hexo g #生成静态网页
hexo d #开始部署

监视文件变动

hexo generate #使用 Hexo 生成静态文件快速而且简单
hexo generate –watch #监视文件变动

完成后部署

两个命令的作用是相同的
hexo generate –deploy
hexo deploy –generate
hexo deploy -g
hexo server -g
